2. **Better task tracking** — Things falling through cracks. Example: `gh-comment-pipeline` was merged but still shows as failed in Slack alerts. Need to fix state reconciliation so merged tasks get properly updated.

3. **Proactive issue tracking** — Monitor `alerts-critical` channel for Sentry/prod issues and auto-dispatch fix agents. Close the gap with Elvis's Sentry scanning.

4. **Battle-test auto-split** — The `auto-split.sh` logic hasn't been tested in a real "needs split" scenario. Need to deliberately trigger and verify it works end-to-end.

5. **Frontend UI testing agent** — Dedicated QA agent that runs Playwright screenshot/visual tests. Gets context from me, does quality assurance on PRs. Combined with the PR comment → update PR loop, this could be very powerful. Potentially a separate agent entirely.

6. **Screenshot CI gate** — Require screenshots for UI-touching changes (like Elvis's setup).

7. **Proactive task discovery** — Morning Sentry scans, meeting note scanning, evening changelog/docs generation from git log. All things Elvis does that we don't yet.

8. **Richer Slack alerts on failures** — When audit or testing triggers a fix cycle, include a short summary of *what* failed in the Slack alert (e.g. "critical: null pointer on guest checkout flow" not just "audit failed"). Extract top findings from audit verdict / test output and embed in the phase transition message.
